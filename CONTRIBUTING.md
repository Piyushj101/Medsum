# Contributing to MEDSUM

Thank you for your interest in contributing to MEDSUM.

MEDSUM is an AI-powered Hospital Management System developed as a collaborative college project. We welcome meaningful contributions that improve the project's functionality, documentation, user experience, or overall code quality.

---

## How to Contribute

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes following the project structure.
5. Test your changes before committing.
6. Write clear and meaningful commit messages.
7. Submit a Pull Request with a brief description of your changes.

---

## Contribution Guidelines

Please ensure that your contribution:

- Follows the existing project structure and coding style.
- Includes clear and readable code.
- Does not break existing functionality.
- Is properly tested before submission.
- Includes documentation updates whenever necessary.

---

## Reporting Issues

If you encounter a bug or have suggestions for improvement, please include:

- A clear description of the issue.
- Steps to reproduce the problem.
- Expected behavior.
- Actual behavior.
- Screenshots or error logs (if applicable).

---

## Commit Message Examples

Good examples:

```
feat: add patient appointment validation

fix: resolve login authentication issue

docs: update project documentation

refactor: improve doctor dashboard API
```

---

## Questions

If you have any questions regarding the project, please contact the project maintainers before making major changes.

---

## Contributors

MEDSUM was developed as a collaborative college project.

Each contributor is responsible for different components of the system, including frontend development, backend development, AI integration, testing, documentation, and project coordination.

Thank you for helping improve MEDSUM.
